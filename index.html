Units of Measurements
Cheat sheet

Unit Description Usage
px A absolute unit that represents a specific number of screen pixels. For example, font-size: 16px; sets the text size to 16 pixels. Since it’s fixed, it doesn’t scale with the screen size, which can sometimes affect responsiveness. This is the most common measurement in design. Best for precise and fixed sizes, such as small UI elements (buttons, borders) or when you need exact control over spacing and font sizes (e.g., font-size: 16px; for consistent text size).
% A relative unit that sets a size based on its parent element. For example, width: 50%; means the element takes up half the width of its parent, making it more flexible for responsive designs. Great for flexible and responsive layouts, such as making an image take up half its container (width: 50%;) or setting a div’s height to fill part of its parent.
em A relative unit based on the font size of the element’s parent. If a parent element has font-size: 16px, then 1em equals 16px, 2em equals 32px, and so on. This allows scaling but can become tricky with nested elements. Useful for scalable typography and spacing. Since it inherits from the parent element, it’s good for buttons, padding, or nested elements (e.g., padding: 1.5em; adjusts proportionally based on the text size).
rem Similar to em, but always relative to the root () font size, which is usually 16px by default. So 1rem is always 16px, making it more predictable than em for consistent scaling across the website. Ideal for maintaining consistent and scalable typography across a website since it always refers to the root font size. Commonly used for text sizes (font-size: 1.2rem;) to ensure uniform scaling.
vh
vw		 vh (Viewport Height) & vw (Viewport Width) are units set sizes based on the size of the browser window. 1vh is 1% of the viewport height, and 1vw is 1% of the viewport width. For example, font-size: 5vw; makes text scale with the screen width. Useful for full-screen designs, such as setting a hero section’s height to 100vh (filling the entire viewport height) or making text scale with screen width (font-size: 5vw;).
vmin
vmax		 vmin & vmax are dynamic units based on the shortest (vmin) or longest (vmax) length of the viewport. 1vmin equals 1% of the smaller dimension (height or width), while 1vmax equals 1% of the larger dimension, making them useful for responsive layouts. Best for responsive elements that need to scale based on screen size. For example, font-size: 3vmin; ensures text scales proportionally on both mobile and desktop without becoming too large or too small.

Table Properties
Cheat sheet

Name Description Usage
<table> The <table> element defines a table structure on a webpage. It wraps all table-related content, including rows and columns. <table> <tr> <td> Data </td> </tr> </table> creates a simple table
<tr> The <tr> element defines a table row. It's used to group a set of table cells (<td> or <th>) within a table. <tr> <td> Row 1, Column 1 </td> </tr> creates one row with a single data cell.
<td> The <td> (table data) element represents a data cell inside a row. It holds the actual content like text, numbers, or images in a table. <td> Apple </td> displays “Apple” in a table cell.
<th> The <th> (table header) element defines a header cell that is typically bold and centered. I's used to describe the content of a column or row, and it helps organize data. <td> <th> Fruit </th> would be a table header cell displaying “Fruit.”
<thead> The <thead> element is used to group the header content in a table. It helps separate header rows from body rows and makes it easier to style. <thead> <tr> <th> Name </th> <thead> Price </th> </th> </thead> creates a header for the table.
<tbody> The <tbody> element groups the main content (body) of a table. It's used to separate table data from the header and footer, making tables easier to manage and style. <tbody> <tr> <td> Apple </td> <td> $1 </td> </tr> </tbody> holds the data rows.
<tfoot> The <tfoot> element groups footer rows in a table. It typically contains summary or calculation data. It's placed at the bottom of the table and helps with accessibility and styling. <tfoot> <tr> <td> Total </td> <td> $10 </td> </tr> </tfoot> adds a footer row to the table.
colspan="#" The colspan attribute in a <td> or <th> element allows the cell to span across multiple columns <td colspan="3"> Merged Cell </td> creates a cell that stretches across three columns.
rowspan="#" The rowspan attribute in a <td> or <th> element allows the cell to span across multiple rows. <td rowspan="2"> Merged Cell <td> creates a cell that spans two rows vertically.
